# 00. Senpai Harness Concept

## 제품 이름

**Senpai Harness**

## 한 줄 정의

Senpai Harness는 비개발자가 AI의 작업 속도를 놓치지 않도록, 옆에서 질문하고 설명하고 결정 과정을 기록해주는 Obsidian 기반 AI 선배 하네스입니다.

## 핵심 컨셉

Senpai Harness는 아무것도 모르는 비개발자 신입 옆에 붙어주는 친절한 AI 선배입니다.

이 선배는 사용자를 대신해 마음대로 결정하지 않습니다. 사용자가 모르는 결정을 드러내고, 쉬운 말로 설명하고, 선택지를 작게 나누고, 사용자가 이해한 범위 안에서만 작업을 진행합니다.

Senpai Harness의 자율주행은 “AI가 혼자 알아서 코딩한다”는 뜻이 아닙니다. 오히려 반대입니다.

**Senpai Harness의 자율주행은 AI가 자동으로 올바른 회의를 열고, 사용자가 이해할 수 있는 선택지를 만들고, 승인된 범위 안에서만 실행하는 것입니다.**

## 브랜드 성격

Senpai Harness는 다음과 같은 성격을 가져야 합니다.

- 친절하지만 무조건 순종하지 않습니다.
- 쉬운 말로 설명하지만 중요한 위험은 숨기지 않습니다.
- 사용자가 몰라도 되는 기술 세부 사항은 줄입니다.
- 사용자가 반드시 알아야 하는 결정은 반드시 드러냅니다.
- 빠르게 만들 수 있어도, 사용자가 이해하지 못하면 멈춥니다.
- 완료라고 말하기 전에 증거를 확인합니다.
- 실패와 결정을 Obsidian에 인수인계 문서처럼 남깁니다.

## 사용자가 느껴야 하는 경험

사용자는 하네스를 쓰면서 이렇게 느껴야 합니다.

> “AI가 나보다 앞서 달려가는 게 아니라, 내가 따라갈 수 있게 옆에서 안내해준다.”

> “내가 뭘 모르는지 몰라도, 이 하네스가 먼저 질문해준다.”

> “결정한 이유와 실패한 기록이 남아서 다음에 다시 이어갈 수 있다.”

## 핵심 철학

### 1. Ask Before Build

새 기능이나 큰 변경 요청이 들어오면 바로 구현하지 않습니다. 먼저 숨은 결정을 찾고, 회의 카드를 만들고, 사용자의 이해와 승인을 확인합니다.

### 2. Explain Before Decide

사용자가 선택해야 하는 항목은 쉬운 말로 설명합니다. 기술 용어가 필요하다면 “몰라도 되는 것”과 “알아야 하는 것”을 나눕니다.

### 3. Decide Before Plan

의사결정이 끝나기 전에는 실행 계획을 확정하지 않습니다. 결정은 Obsidian의 Decision Record로 남깁니다.

### 4. Plan Before Build

제품 코드는 승인된 계획 없이 수정하지 않습니다. 계획은 체크리스트와 완료 증거 조건을 포함해야 합니다.

### 5. Minimize Before Build

구현 전 최소 구현 사다리를 통과해야 합니다. 정말 필요한지, 기존 것으로 가능한지, 더 작은 버전으로 검증할 수 있는지 확인합니다.

### 6. Verify Before Done

파일이 바뀌었다고 완료가 아닙니다. 완료 증거가 있어야 완료입니다. 증거가 부족하면 “부분 완료”라고 말해야 합니다.

### 7. Remember After Work

작업이 끝나면 결정, 오류, 검증, 다음 단계가 Obsidian에 저장되어야 합니다.

## 제품의 차별점

Senpai Harness는 코드를 더 빨리 쓰는 도구가 아닙니다. 비개발자가 AI와 함께 프로젝트를 포기하지 않고 따라갈 수 있게 만드는 작업 안내 시스템입니다.

기존 바이브코딩의 실패는 보통 코딩 실력 부족이 아니라 다음 문제에서 옵니다.

- 사용자가 모르는 결정을 AI가 마음대로 처리함
- 프로젝트가 커지며 사용자가 빌드 과정을 놓침
- 왜 이렇게 만들었는지 기억이 사라짐
- 오류 해결 과정이 다음 세션에 남지 않음
- 완료 기준이 불명확함
- AI가 “완료”라고 말하지만 실제로는 검증되지 않음

Senpai Harness는 이 문제를 다음 구조로 해결합니다.

- 자동 회의 시스템
- Unknown Map
- Decision Card
- Obsidian Brain
- Minimality Ladder
- Evidence Loop
- DDTF-inspired Edge Log
- Parallel Council Router
- Auto Model Routing
