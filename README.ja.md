誰にでも初心者だった頃があります。あの頃、先輩からよく言われた言葉があります。「わからないことは聞きながらやって。でも、こんなことまで一つひとつ教えなきゃいけないの？」

Senpai Harness は、わからなそうなことを先に聞いてくれて、決まった内容を親切に教えてくれてから作業に入ります。現実にはいなかった、そんな理想の先輩の役割を、Senpai Harness が果たしてくれます。

バイブコーディングを初めて触れる非開発者にとって、いちばん親切な先輩です。

Senpai Harness は、先に尋ね、わかりやすく説明し、一緒に決め、**承認された範囲だけ**を作り、すべての決定と失敗を Obsidian（Markdownメモアプリ）に記録するAI先輩ハーネスです。ターミナルで使うAIコーディングツール、Claude Code にインストールするプラグインです。

## 誰のためのものか

こんな人たちのために作りました。コーディングの文法を何も知らない人、ターミナルや Claude Code を今日初めて開く人、バイブコーディング（会話するようにAIにコードを任せる方法）でアプリやチャットボット、自動化ツールを作ってみたい人、そしてAIが作り出す結果についていけずプロジェクトを諦めてしまったことがある人。

すでにコーディングに慣れている場合、このハーネスが求める「先に尋ねて承認を得る」手順はかえって遅く感じられるかもしれません。Senpai Harness はスピードより、**最後までついていけること**を目標に作られています。

## 「自動運転」とはどういう意味か

Senpai Harness をインストールすると、AIが勝手にコードを書き始めると思いがちですが、実際にはその正反対です。

ここでの「自動運転」とは、AIが自分だけで判断してコーディングするという意味ではなく、AIが自動的に適切な会議を開き、ユーザーが知らない決定事項を理解しやすい選択肢として提示し、**ユーザーが承認した範囲内だけ**でコードを書くという意味です。

> マンションのリフォーム工事を思い浮かべてください。施主が見積書にサインする前は、どれだけ腕のいい職人でも壁に釘一本打つことはできません。サインは職人が代わりにできませんし、見積書にない部屋はサインがあっても手を付けられません。

承認の文言をチャットで直接送るまでは、AIのファイル書き込みがブロックされます。これはAIが「自分から気をつけている」のではなく、別の安全装置が機械的に強制している仕組みです。

## 何をしてくれるのか

### 隠れた決定事項を先に見せる

> 「ログイン機能だけ付けてください」というお願いは簡単そうに見えますが、実際には壁の中の配線や配管のような隠れた決定事項がついてきます —— 情報をどこに保存するか、パスワードをどう守るか、そもそもログインが本当に必要か。優れた施工業者はいきなり金づちを持たず、まず隠れた問題のリストを見せて、どうするか尋ねます。

Senpai Harness も同じです。「機能を付けて」という依頼が来ても、すぐにコーディングを始めるのではなく、隠れた決定事項をリストにまとめ、A/B/C/D のような選択肢カードとして見せます。より小さな方法で十分な場合は、その代案を**先に**勧めます。

### リスクのある依頼には複数の視点が同時に集まる

> 病院では大きな手術の前に、内科・外科・麻酔科の医師が同時に集まり、それぞれの視点で検討するカンファレンスを行います。カンファレンスの医師たちは意見を出すだけで、メスを握るのは執刀医一人だけです。

「決済機能を入れて」のようなリスク信号のある依頼には、リスク担当・企画担当・検証担当のような複数の役割が同時に検討します。これらのレビュー担当はすべて読み取り専用なので、誰もコードを直接書くことはできません —— メスは依然として一人だけが握っています。

### AIの「終わりました」をそのまま信じない

> 職人が「工事が終わりました」と言っても、監理者（工事がきちんとできているか確認する人）が現場を直接確認するまでは、残金は支払われません。

Senpai Harness は完了を5段階（「部分完了」から「検証完了」まで）でしか報告させず、根拠なく「完了」とだけ言うことを防ぎます。「終わった?」と聞かれれば、まだできていない部分はできていないと**正直に**答えます。

### 決定事項と失敗を次のセッションに引き継ぐ

> 担当者が変わっても仕事が途切れない会社には、引き継ぎノートがあります。何をなぜ決めたのか、どんなトラブルがあってどう収拾したのかが書かれているので、新しい担当者が最初から把握し直す必要がありません。

Senpai Harness は決定した理由、エラーを解決した過程、今日行った作業と残っている作業を Obsidian の Vault（保管フォルダ）に自動的に記録します。次回「続きをお願い」と言うだけで、前回どこまで進んだかから改めて見せてくれます。

## インストール

Claude Code で以下の3行を順番に入力してください。

```text
/plugin marketplace add calmtiger86/senpai-harness
/plugin install senpai-harness@senpai-harness
/senpai-harness:init
```

最後の行まで実行すると、プロジェクトに `vault/`（Obsidian の保管フォルダ）、`CLAUDE.md`、`AGENTS.md`、`senpai.config.yaml` が作成されます。このうち `senpai.config.yaml` がハーネスの安全装置をオンにするスイッチです —— このファイルがないプロジェクトでは一切の検査が行われず、Claude Code のデフォルトの動作のまま動きます。

インストールや動作中に問題があれば `/senpai-harness:doctor` を入力してみてください。何が問題なのか、開発者でなくても読める言葉で教えてくれます。

新しいバージョンの内容が気になる場合は [`CHANGELOG.md`](CHANGELOG.md) を確認してください。
