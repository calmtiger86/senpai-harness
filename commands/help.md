아래 내용을 비개발자가 이해할 수 있는 말로, 사용자의 실제 상황에 맞게 골라서 설명합니다. 전문 용어를 쓸 때는 짧게 풀어줍니다.

## Senpai Harness가 하는 일

명령어를 몰라도 됩니다. 그냥 하고 싶은 말을 하면 됩니다.

- "새 앱 만들고 싶어" → 먼저 몇 가지를 물어보고 방향을 정합니다.
- "로그인 기능 붙여줘" → 바로 만들지 않고, 숨어 있는 결정(로그인 방법, 저장 방식 등)을 먼저 보여줍니다.
- "어제 하던 거 이어서 해줘" → 지난 기록을 불러와 이어갑니다.
- "다 됐어?" → 실제로 확인한 뒤에만 "완료"라고 답합니다(증거 없이 대충 답하지 않습니다).
- "오늘 여기까지" → 지금까지 한 일을 기억에 저장하고 정리합니다.

## 승인이 필요한 순간에 쓰는 문구 (정확히 그대로 입력해야 함)

- `[senpai-go:프로젝트이름]` — 계획(Phase Plan)을 검토한 뒤, 그 범위 안의 작업을 승인합니다.
- `[senpai-touch:프로젝트이름:파일경로]` — 특히 민감한 파일(로그인, 결제 등) 하나를 개별로 다시 승인합니다.
- `[senpai-stop]` — 이 프로젝트에서 Senpai Harness 관리를 끕니다(기록은 남습니다).

이 문구들은 일부러 딱딱합니다 — "네", "그렇게 해줘" 같은 말로는 승인되지 않습니다. AI가 스스로 승인 권한을 확장하지 못하게 막기 위한 장치입니다.

## 슬래시 커맨드 (선택 사항, 몰라도 대화로 다 됩니다)

- `/senpai-harness:init` — 이 프로젝트에 처음 설치.
- `/senpai-harness:doctor` — 설치 상태 점검.
- `/senpai-harness:status` — 지금 프로젝트 상태 보기.
- `/senpai-harness:recall` — 마지막 세션 기록 불러오기.
- `/senpai-harness:save` — 지금까지 진행 상황 중간 저장.
- `/senpai-harness:reset` — 관리 중단 방법 안내.
- `/senpai-harness:help` — 지금 이 도움말.

## 알아두면 좋은 것

- 승인 전에는 제품 코드를 수정하지 않습니다.
- 완료 증거 없이 "완료했습니다"라고 말하지 않습니다.
- 중요한 결정과 오류는 이 프로젝트의 `vault/` 폴더(Obsidian 볼트)에 기록으로 남습니다.
