Senpai Harness 是一个陪伴初次尝试氛围编程（Vibe Coding，即像聊天一样把编码工作交给 AI 的方式）的人的 AI 前辈框架：先询问、用简单的语言解释、和你一起做决定、**只在获得批准的范围内**动手，并把每一次决定和失败都记录到 Obsidian（一款 Markdown 笔记应用）中。它以插件形式安装在 Claude Code（一款终端 AI 编程工具）上。

## 适合谁使用

这是为以下这些人打造的：完全不懂编程语法的人、今天第一次打开终端或 Claude Code 的人、想通过氛围编程（用对话的方式把代码交给 AI 完成）做出应用、聊天机器人或自动化工具的人，以及曾经因为跟不上 AI 的进度而放弃过项目的人。

如果你已经很熟悉编程，这个框架要求的"先询问、再获批准"流程可能会让你觉得慢。Senpai Harness 追求的不是速度，而是**让你能够跟到最后**。

## "自动驾驶"到底是什么意思

安装 Senpai Harness 之后，很容易让人以为 AI 会开始自行编写代码 —— 但实际情况恰恰相反。

这里所说的"自动驾驶"，并不是指 AI 自己判断、自己写代码，而是指 AI 会自动召开合适的会议，把你原本不知道需要做的决定，转化成你能理解的选项呈现出来，并且**只在你已批准的范围内**写代码。

> 想象一下装修房子。在业主在报价单上签字之前，无论工人的手艺多好，都不能在墙上钉一颗钉子。签字这件事工人不能代劳，而且哪怕签了字，报价单上没写的房间也不能动。

在你亲自通过聊天发送批准指令之前，AI 的文件写入功能会一直被阻止。这不是 AI"自己小心谨慎"，而是由一套独立的安全机制在每一次都强制执行的结果。

## 它能做什么

### 先把隐藏的决定摆在你面前

> "帮我加个登录功能"听起来很简单，但实际上牵涉到墙内布线和管道一样隐藏的决定：数据存在哪里、密码怎么保护、登录功能是否真的有必要。优秀的施工方不会先拿起锤子，而是先列出隐藏问题清单，问你打算怎么处理。

Senpai Harness 也是如此。当你提出"加个功能"的请求时，它不会立刻开始写代码，而是先把隐藏的决定整理成清单，以 A/B/C/D 选项卡的形式呈现给你。如果一个更小的方案就已经够用，它会**优先**推荐那个方案。

### 高风险请求会同时汇集多个视角

> 大型手术前，内科、外科、麻醉科医生会同时集合进行会诊，各自从自己的角度审查。会诊医生只提供意见 —— 手术刀始终只握在主刀医生一人手中。

像"加个支付功能"这样带有风险信号的请求，会同时由风险把关、产品规划、验证核查等多个角色一起审查。这些审查者全部是只读的，谁都不能直接写代码 —— 手术刀依然只握在一个人手中。

### 不会照单全收 AI 自称的"完成了"

> 即便工人说"工程完工了"，在监理（负责确认工程是否真正做好的人）亲自到现场核实之前，尾款也不会支付。

Senpai Harness 只允许用五个明确的等级来汇报完成情况（从"部分完成"到"已验证完成"），并且禁止在没有证据的情况下直接宣称"完成"。当你问"做完了吗？"，它会**诚实地**告诉你哪些部分还没做完。

### 把决定和失败经验带到下一次会话中

> 一家即便负责人更换、工作也能顺利延续的公司，会有交接笔记：记录了做了什么决定、为什么这样决定，发生过什么事故、又是如何处理的 —— 这样新负责人就不必从零开始重新摸索。

Senpai Harness 会自动把决定的理由、错误的解决过程、今天完成的工作和剩余的工作，记录到 Obsidian 笔记库（记忆文件夹）中。下次只要说一句"继续"，它就会重新展示上次进行到哪里了。

## 安装

在 Claude Code 中依次输入以下三行命令。

```text
/plugin marketplace add calmtiger86/senpai-harness
/plugin install senpai-harness@senpai-harness
/senpai-harness:init
```

执行完最后一行后，项目中会生成 `vault/`（Obsidian 记忆文件夹）、`CLAUDE.md`、`AGENTS.md`、`senpai.config.yaml`。其中 `senpai.config.yaml` 是打开框架安全机制的开关 —— 没有这个文件的项目不会运行任何检查，行为和原生 Claude Code 完全一样。

如果在安装或使用过程中遇到问题，可以输入 `/senpai-harness:doctor`，它会用非开发者也能看懂的语言告诉你问题出在哪里。

想了解最新版本的更新内容，请查看 [`CHANGELOG.md`](CHANGELOG.md)。
