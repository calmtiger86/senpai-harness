# Unknown Detector

Unknown Detector는 사용자가 모르는 숨은 결정을 찾아내는 역할을 합니다. 겉보기에는 버튼 하나, 기능 하나처럼 보여도 실제로는 여러 결정이 함께 딸려 오는 경우가 많습니다. 이 문서는 무엇을 확인해야 하는지 10가지 범주로 정리한 체크리스트입니다.

Discovery Meeting이 열릴 때, 이 체크리스트를 기준으로 [[Schema|Unknown Map]] 노트를 채웁니다.

## 확인 체크리스트

- [ ] **제품 방향** — 이 기능이나 요청이 전체 제품의 목적과 방향에 맞는지 확인했는가
- [ ] **사용자 흐름** — 사용자가 실제로 어떤 순서와 화면으로 이 기능을 쓰게 되는지 정리했는가
- [ ] **데이터 저장** — 무엇을 어디에, 어떤 형태로 저장할지(로컬 저장인지, 서버 저장인지 등) 정했는가
- [ ] **로그인/인증** — 로그인이 필요한지, 필요하다면 이메일 로그인인지 소셜 로그인인지 정했는가
- [ ] **결제** — 결제가 관련되는지, 관련된다면 어떤 안전장치가 필요한지 확인했는가
- [ ] **개인정보** — 개인정보를 다루는지, 다룬다면 어떻게 보호하고 어디까지 수집할지 정했는가
- [ ] **플랫폼 제한** — 웹/모바일/OS 등 사용 환경에 따른 제약을 확인했는가
- [ ] **배포** — 언제, 어떤 방식으로 배포할지(또는 아직 배포하지 않을지) 정했는가
- [ ] **유지보수** — 나중에 누가, 어떻게 이 기능을 고치고 관리할지 생각해봤는가
- [ ] **검증 기준** — 이 기능이 "완료"되었다고 말할 수 있는 기준이 무엇인지 정했는가

10가지 범주를 모두 확인할 필요는 없지만, 확인하지 않고 넘어간 범주가 있다면 그 이유를 사용자에게 말해야 합니다.

## 산출물

Unknown Detector가 회의를 마치면 다음을 만들어 냅니다.

- **Unknown Map** — 위 범주별로 아직 모르는 것을 정리한 노트 (`10_Projects/{project}/Unknown Map.md`)
- **hidden_decisions** — 사용자가 모르고 있었던 결정 목록
- **risk_candidates** — 위험할 수 있는 지점 후보
- **decision_questions** — 사용자에게 물어야 할 질문 목록 (Decision Card로 이어짐)

## 이 체크리스트를 쓰는 이유

바이브코딩이 실패하는 가장 흔한 이유는 코딩 실력 부족이 아니라, 사용자가 모르는 결정을 AI가 마음대로 처리해버리기 때문입니다. 이 체크리스트는 그 결정들을 미리 드러내서, 사용자가 이해한 범위 안에서만 작업이 진행되도록 만듭니다.
