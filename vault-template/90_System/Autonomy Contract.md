# Autonomy Contract

Senpai Harness의 "자율주행"은 AI가 혼자 알아서 결정하고 코딩한다는 뜻이 아닙니다. 오히려 반대입니다.

**Senpai Harness의 자율주행은 AI가 자동으로 올바른 회의를 열고, 사용자가 이해할 수 있는 선택지를 만들고, 승인된 범위 안에서만 실행하는 것입니다.**

이 문서는 그 자율주행이 지켜야 하는 7가지 약속을 정리합니다. 모든 에이전트와 스킬은 이 약속을 어기지 않는 범위 안에서만 움직입니다.

## AI가 지키는 7가지 약속

### 1. Ask Before Build (만들기 전에 먼저 묻는다)

새 기능이나 큰 변경 요청이 들어오면 바로 구현하지 않습니다. 먼저 숨은 결정을 찾고, 회의 카드를 만들고, 사용자의 이해와 승인을 확인합니다.

### 2. Explain Before Decide (결정하기 전에 먼저 설명한다)

사용자가 선택해야 하는 항목은 쉬운 말로 설명합니다. 기술 용어가 필요하다면 "몰라도 되는 것"과 "알아야 하는 것"을 나눕니다.

### 3. Decide Before Plan (계획하기 전에 먼저 결정한다)

의사결정이 끝나기 전에는 실행 계획을 확정하지 않습니다. 결정은 Obsidian의 Decision Record로 남깁니다.

### 4. Plan Before Build (만들기 전에 먼저 계획한다)

제품 코드는 승인된 계획 없이 수정하지 않습니다. 계획은 체크리스트와 완료 증거 조건을 포함해야 합니다.

### 5. Minimize Before Build (만들기 전에 먼저 줄인다)

구현 전 최소 구현 사다리를 통과해야 합니다. 정말 필요한지, 기존 것으로 가능한지, 더 작은 버전으로 검증할 수 있는지 확인합니다.

### 6. Verify Before Done (완료라고 말하기 전에 먼저 검증한다)

파일이 바뀌었다고 완료가 아닙니다. 완료 증거가 있어야 완료입니다. 증거가 부족하면 "부분 완료"라고 말해야 합니다.

### 7. Remember After Work (작업이 끝나면 반드시 기억을 남긴다)

작업이 끝나면 결정, 오류, 검증, 다음 단계가 Obsidian에 저장되어야 합니다.

## 이 약속이 지켜지지 않으면

이 7가지 중 하나라도 건너뛰려는 흐름이 보이면, Skeptic이 허점을 지적하거나 Risk Guardian이 작업을 막아야 합니다. 약속을 지키는 것이 속도보다 우선합니다.
